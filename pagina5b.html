[6] Het DOM model

Fundamenteel voor de mogelijkheid om binnen Javascript heel gericht onderdelen van een webpagina te kunnen beïnvloeden is het DOM model, het Document Object Model. Dit model geeft aan hoe een webbrowser intern een pagina bekijkt, welke objecten (alinea’s, formulieren, koppen, div’s) er op de pagina staan en in welke hierarchie deze objecten staan. Dit is onder andere van belang voor hoe css opmaak wordt doorgegeven. Maar Javascript maakt er dus ook gebruik van om attributen van html objecten mee te beinvloeden. DOM objecten hebben, zoals het echte objecten betaamt, attributen, maar ook methoden.

Het DOM Model is in de kern niets anders dan een representatie van de elementen van een webpagina:

Ieder punt in deze ‘boom’ heet een node. Er zijn drie soorten nodes:

  • element nodes (in feite HTML tags, die ‘kinderen’ (=andere nodes binnen zichzelf) en ‘attributen’ kunnen hebben) In het voorbeeld hierboven bijvoorbeeld div, maar er zijn er meer.
  • text nodes (die alleen maar uit tekst bestaan (geen tags dus) In het voorbeeld hierboven title.
  • attribuut nodes (die een attribuut van het element benoemen) In het voorbeeld hierboven kan de div een class of id attribuut hebben.
Deze <a> tag bijvoorbeeld: <a href=”test.html”>Klik hier</a> is zelf een element node, heeft als tekst ‘klik hier’ en als ‘link’ attribuut: test.html.

Tags binnen een tag kunnen allerlei aanduidingen krijgen (‘firstChild, nextSibling,parentNode’) die duidelijk maken waar zij zich in de hierarchie bevinden. Ook kunnen met de methoden als appendChild(nod) en removeChild(node) nodes aan de boom gemanipuleerd worden.

Nodes kun je met Javascript benaderen via commando’s als 


document.getElementById(“id”)
document.getElementsByTagName(“tag”)
Bij de eerste mik je op één ding, bij het tweede kun je alle <p> tags op een pagina benaderen.

In een Javascript programma kun je verschillende globale objecten benaderen:

  • window = het browser scherm
  • navigator = informatie over de webbrowser die je gebruikt (welke is het, mogen cookies? enz.)
  • screen = informatie over de monitor (beschikbare resolutie e.d.)
  • history = de pagina’s die de browser bezocht heeft (ook methodes als back en go)
  • location = de URL van de huidige pagina (host, protocol)
  • document = het DOM model van de huidige pagina.
In het eerder gegeven voorbeeld: 

var melding = “Goedenacht!”;
document.getElementById(“melding”).innerHTML = melding;
zie je dus het DOM model in actie. Op de pagina wordt gezocht naar het element dat een id “melding” heeft. Van dit element wordt de html in de tag zelf veranderd in de inhoud van de variabele melding.
