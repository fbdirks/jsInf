5. Hoe praat JS met je?

De interface van een JS programma wordt verzorgd door HTML, bijvoorbeeld door een formulier en door een juiste naam- en id-geving van elementen op de pagina. Via het DOM model kan JS bij ieder HTML/CSS element dat op de juiste manier te identificeren valt de eigenschappen of waarden beïnvloeden. Zie de paragraaf over het DOM model.

Hier zie je vier (vijf eigenlijk) manieren waarop Javascript zijn uitvoer kwijt kan:

1: Veranderen van de inhoud van HTML elementen op de pagina

Hier verandert Javascript de HTML code die wordt omsloten door een specifieke tag: 

var melding = “Goedenacht!”;
document.getElementById(“melding”).innerHTML = melding;

In het HTML deel van deze pagina moet dan iets dergelijks staan:

2: veranderen van de complete inhoud van een pagina

Daarnaast kan de JS output door de browser als een complete webpagina worden opgevat (document.write()). 
var melding = “Goedemorgen!”;
document.write(melding);
Omdat dit dus de complete inhoud van een browserpagina verandert is het maar beperkt nuttig.

3: Uitvoer van informatie naar een popup scherm

Via een alert() kan er ook een popup scherm worden getoond. 

var melding = “Goedemiddag!”;
alert(melding);
Pop-ups zijn bij gebruikers niet zo populair. In de browsers kun je pop-ups meestal ook uitzetten.

4: Uitvoer naar de console

Via console.log() kan er geschreven worden naar de Javascript console die in een browser geopend kan worden (bijvoorbeeld met F12). Dit is meer iets voor ontwikkelaars. Deze vorm van output is niet geschikt voor gewone gebruikers die een webpagina willen bekijken. 

var melding = “Goedeavond!”;
console.log(melding);

5: Uitvoer naar het werkscherm van een andere applicatie

Als je Javascript gebruikt voor het automatiseren van applicaties zul je in de documentatie van deze applicaties moeten kijken naar de manier waarop je bij die applicatie output daar kunt krijgen waar je wilt. Dit is een voorbeeld van UltraEdit (een programmeurs teksteditor). 


var melding = "Ook goededag!";
UltraEdit.activeDocument.write(melding);
Dit is een mooi voorbeeld van een objectbibliotheek die wel verschilt van Javascript maar er toch ook weer heel erg op lijkt.
