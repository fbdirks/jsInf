[17] “Alles is een object”

Javascript is object georiënteerd. Dat is niet altijd meteen duidelijk. Je kunt naar hartelust procedureel en functioneel aan de gang zonder over objecten na te hoeven denken. Maar het browserscherm waarin je met javascript allerlei dingen doet is al een object dat kenmerken en methoden heeft. Een browserscherm hoort bij het object window. Eigenschappen van dit object zijn bijvoorbeeld de naam van het window, de positie op het scherm, de locatie (dus het adres van de pagina die getoond wordt), de geschiedenis en het protocol waarmee de pagina geopend is. De methoden van dit object zijn onder andere

  • print (waarmee je de printdialoog voor een browserscherm start)
  • alert (waarmee je een mededeling kunt openen)
  • confirm (waarmee je een bevestigingsvraag kunt stellen)
  • open (waarmee je een nieuwe of vervangende pagina kunt laden)
  • back (waarmee je in de geschiedenis terug kunt gaan)
Gebruik maken van kenmerken en methoden kan met behulp van de dot notatie. Deze programma regel bijvoorbeeld zet in de variabele hoogte de binnenhoogte van het browserscherm: 
var hoogte = window.innerHeight ;
Deze regel springt een pagina terug in de geschiedenis: 
window.history.back();
en deze regel start het uitprinten van een pagina: 
window.print();
Bij doorverwijzen heb je vaak window.open nodig. Dit is een code fragment waarin ook allerlei parameters van de te openen pagina worden gezet: 
var myWindow = window.open("https://www.w3schools.com", "cursus", "width=200,height=100");

Documentatie is onontbeerlijk als je met het window object aan de gang wilt gaan. Je vindt deze hier: developer.mozilla.org/en-US/docs/Web/API/Window .

Het is belangrijk om met dit soort objecten te leren stoeien. We zullen later zien dat er allerlei javascript bibliotheken zijn die je van groot nut kunnen zijn omdat ze of de omgang met de DOM vergemakkelijken, of jou de noodzaak uit handen nemen om voor iedere denkbare browserversie die er maar bestaat of bestaan heeft een functie te schrijven die werkt. Dit soort bibliotheken zijn vaak opgezet in die object vorm die ook window kenmerkt. Hoe meer je met die manier van werken thuis bent, hoe sneller je met die bibliotheken aan de slag kunt.
