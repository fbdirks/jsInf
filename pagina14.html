Keuzes maken: switch

Als je heel veel verschillende situaties moet vergelijken met een variabele is een switch commando soms handiger dan alsmaar if/elseif/elseif/elseif/else . Het leest prettiger en is minder foutgevoelig.

Dit is een Switch voorbeeldje: 

   
switch (new Date().getDay()) {
    case 6:
    	text = "Today is Saturday";
        Break;
    case 5:
    	text = "Today is Friday";
        Break;
    case 4:
    	text = "Today is Thursday";
        Break;

    // enzovoorts

    case 0:
    	text = "Today is Sunday";
        break; 
    default: 
    	text = "Looking forward to the Weekend";
}

In de ronde haken van de switch regel staat op welke variabele gelet zal worden. In het voorbeeld is dat de uitkomst van de javascriptfunctie die bekijkt welke dag het vandaag is. Achter de diverse 'case' regels staat de waarde van die variabele die een bepaalde actie vraagt. Die actie staat achter de : van de case regel genoteerd. Het break commando is nodig om na het uitvoeren van die acties de switch te stoppen. Het is niet altijd nodig, maar voor de zekerheid kan het geen kwaad altijd een case blok met een break te eindigen. Het blok dat begint met default: is het 'afvoerputje': als de variabele aan geen van de 'case'gevallen voldoet zullen de acties die bij default staan uitgevoerd worden.
