[18] Objectgeoriënteerd Programmeren in Javascript

Objectgeoriënteerd Programmeren in Javascript werkt conceptueel hetzelfde als in PHP, maar de manier waarop je het in code vormgeeft is nogal anders.

Een object maken is eenvoudig. De volgende code regel maakt een leeg object aan dat mijnObject heet. 

mijnObject = { } ;
Eigenschappen kun je op deze twee verschillende manieren aan een object toevoegen (beide manieren doen hetzelfde): 

mijnObject.naam = “zomaar iets”;
mijnObject[‘kleur’] = “rood”;
De tweede methode is bijvoorbeeld handig om alle objecten van een methode te laten zien: 
for (a in mijnObject) {
	console.log(a + mijnObject[a]);
}
Dit kan beslist niet met de eerste methode.

Methoden kun je ook toevoegen aan een object door er een anonieme functie aan toe te kennen, op deze manier: 

mijnObject.tonen = function() { 
	console.log(this.naam);
}
De functie zelf heeft geen naam, maar je kent hem toe aan een methodenaam in een object. Deze techniek die voor wie het anders gewend is soms even wennen.

Het is ook mogelijk om eigenschappen en attributen in één keer in een soort object model te gieten: 

function mijnObject(naam, kleur) {
	this.naam = naam || “onbekend”,
	this.kleur = kleur || “onbenoemd”,
	this.tonen = function() {
		console.log(this.naam);
	}
}
Let op een paar kleinigheden die belangrijk zijn: iedere eigenschap/methode wordt van de anderen gescheiden door een KOMMA, en niet een PUNTKOMMA. En op de tweede regel van de objectdefinitie zie je een voorbeeld van een defaultwaarde die toegekend wordt als een parameter (bijvoorbeeld naam) ‘false’ (= niet ingevuld) is.

Deze functie werkt pas als je op deze manier een objectinstantie aanmaakt: 

g = new mijnObject();
g.tonen(); // ‘onbekend’omdat de naam in de declaratie ontbrak

h = new mijnObject(“zomaar een waarde”);
h.tonen(); // ‘zomaar een waarde’;

Voor gevorderden en liefhebbers: prototypen van klassen

Het nadeel van de objectdefinitie die hierboven gebruikt wordt is dat methoden die helemaal geen nieuwe waarden geven aan attributen bij ieder nieuw object apart in het geheugen worden opgeslagen. Onnodig, want ze zijn voor ieder object 100% hetzelfde. Je kunt dat voorkomen door voor een object een prototype te benoemen. Op deze manier: 

mijnObject.prototype.tonen() {
	console.log(this.naam + “  “ + this.kleur);
};
Deze methode mag dan niet opduiken in de ‘normale’ objectdefinitie. Als in runtime voor een concreet object de methode tonen() wordt aangeroepen kijkt Javascript eerst of het voor dat object definieerd is, en als dat niet zo is of het in het prototype van het object voorkomt. Door het gebruik van this in de definitie van de prototype definitie wordt voor de concrete attributen de concrete waarde van een object ingevuld. Die bestaan natuurlijk niet in een prototype, maar dat hoeft ook niet.

Het complete voorbeeld wordt dan: 

function mijnObject(naam,kleur) {
	this.naam = naam || "onbekend",
  this.kleur = kleur || "onbenoemd"
}

/* de prototype methode */
mijnObject.prototype.tonen = function() {
    console.log(this.naam + " " + this.kleur);
}

/* de objecten aan maken */
g = new mijnObject();
g.tonen();

h = new mijnObject("zomaar een waarde");
h.tonen(); // ‘zomaar een waarde’;
